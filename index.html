📜 第二章：AI 前世今生

不需要深入，但要知道来龙去脉。这样听到术语不会懵。

🕰️1950s-2010s：传统AI —— 规则和统计的时代▶

1950年，图灵提出"机器能思考吗？"——这是AI的起源。

之后的几十年，AI主要有两条路：

  • 符号主义：人写规则，机器执行。比如"如果体温>37.3°C，则标记为发热"。问题：规则写不完。
  • 统计学习：让机器从数据中找规律。比如给机器看1000张猫的照片，它学会了识别猫。问题：需要人工提取特征。

这个时代的AI很"笨"，只能做很窄的任务。你听到的专家系统、决策树、SVM都是这个时代的产物。

🔥2012年：深度学习爆发 —— 机器自己学特征▶

2012年，AlexNet在图像识别比赛中碾压传统方法，标志着深度学习时代到来。

深度学习的突破在于：不需要人工提取特征了，神经网络自己从数据中学习特征。

关键概念：

  • 神经网络：模拟人脑神经元的计算模型，一层一层地处理数据
  • 深度学习：很多层的神经网络（"深"就是层数多）
  • GPU加速：GPU天生适合做大量并行计算，深度学习必须靠GPU
  • 训练：用大量数据调整神经网络的参数，让它学会某种能力
⚡2017年：Transformer —— 大模型的"基石"▶

2017年，Google发表论文《Attention Is All You Need》，提出了Transformer架构。

Transformer的核心是注意力机制（Attention）：让模型在处理每个词时，自动"关注"最相关的其他词。

简单理解：读一句话"小明养了一只猫，他很喜欢它"，注意力机制让模型知道"它"指的是"猫"而不是"小明"。这就是上下文理解。

Transformer为什么重要？因为所有现代大模型（GPT、通义千问、DeepSeek、ChatGLM）都是基于Transformer构建的。它是大模型的"祖宗"。

🧠2018-2020：预训练模型 —— 先学通识再学专业▶

2018年，Google推出BERT；2020年，OpenAI推出GPT-3。

核心思路：预训练 + 微调

  • 预训练：用互联网上所有能找到的文字（万亿级token）训练模型 → 学会语言的通用规律 → 这一步极其昂贵（几百万到几千万美元）
  • 微调：用特定领域的数据（如公安卷宗）再训练 → 学会专业术语和逻辑 → 这一步成本较低
⚠️ 你听到的"大模型"就是预训练后的产物。它像一个读了全人类所有书的超级毕业生——什么都懂一点，但不精通任何专业。微调就是送它去"公安大学"进修。
🚀2022至今：大模型时代 —— ChatGPT引发的革命▶

2022年11月，ChatGPT发布，两个月用户破亿。大模型从一个技术名词变成了全民话题。

之后发生的事：

  • 2023年：开源大模型爆发（LLaMA、Qwen、ChatGLM、DeepSeek）
  • 2023年：RAG技术成熟，解决大模型"幻觉"问题
  • 2024年：智能体（Agent）成为热点，AI不只是聊天，还能执行任务
  • 2024年：Dify等低代码平台出现，让非程序员也能开发智能体
  • 2025年：超级智能体（Skill-based Agent），AI可以自动编排复杂工作流
✅ 到这里你就知道：我们现在处于"智能体时代"。大模型是基础，RAG是增强，智能体是应用，Dify是开发工具。你学的所有东西都在这条线上。

📖 第三章：核心名词渐进式解读

按依赖关系排列，先学前面的才能理解后面的。每个名词：一句话解释 + 会议中的应用 + 类比

🧠大模型（LLM） —— 一切的基础▶

一句话：读了互联网上海量资料后训练出来的"超级大脑"，能读懂人话、写报告、回答问题。LLM = Large Language Model。

会议中："怎么把这个千问用好"——千问(通义千问/Qwen)就是你们部署的大模型。

类比：一个读过全世界所有书的超级毕业生，你问他什么都能答上一点，但可能答不准。

关键数字：模型大小用参数量衡量，7B = 70亿参数，72B = 720亿参数。参数越多越聪明，但也越吃配置。

国产常用：

模型出品方可本地部署特点
通义千问(Qwen)阿里✅你们在用的
DeepSeek深度求索✅推理能力强
ChatGLM智谱AI✅轻量版可消费级GPU
💭幻觉（Hallucination） —— 大模型最大的毛病▶

一句话：大模型一本正经地胡说八道，编造不存在的事实。

为什么会有幻觉：大模型本质上是"预测下一个字"的机器，它不是查资料回答的，而是"猜"出来的。猜对是知识，猜错就是幻觉。

类比：让一个只读过书但没上过法庭的法学毕业生审卷宗，他可能说得头头是道，但细节全是编的。

⚠️ 公安场景下幻觉极其危险！AI可能编造不存在的法律条文，或者"发现"实际不存在的案件问题。所以所有AI输出必须人工复核。

怎么解决？→ 下一个词：RAG

📚RAG（检索增强生成） —— 让AI"开卷考试"▶

一句话：先让AI从知识库里查资料，再基于查到的资料回答问题，而不是自己瞎编。

RAG = Retrieval-Augmented Generation

流程：用户提问 → 系统从知识库检索相关内容 → 把检索结果+问题一起给大模型 → 大模型基于真实资料回答

会议中："走RAG flow的那种向量知识"、"知识召回"——就是RAG检索。

类比：闭卷考试（裸大模型）vs 开卷考试（RAG）。开卷考试不一定能考满分，但至少不会编造不存在的答案。

✅ 你们做的卷宗查错、法规检索，核心架构都是RAG。
🔢向量化（Embedding） + 向量数据库 —— RAG的底层引擎▶

Embedding一句话：把一段文字转换成一串数字（向量），让计算机能计算"这两段话意思有多像"。

向量数据库一句话：专门存这些向量数字的数据库，支持"按意思相近"搜索，而不是按关键词搜索。

为什么RAG需要它：民警问"治安案件办案期限多长"，传统数据库搜不到（因为"期限"≠"时间"≠"多久"），但向量数据库能找到"办案时限30日"那条（因为它俩"意思相近"）。

会议中："走一范的那种向量知识"、"建立知识库的方法，成本最低的还是走向量"——就是向量数据库。

常用向量数据库：

名称特点场景
Milvus开源，功能全大规模生产
Chroma轻量，易上手快速原型
RAGFlow专门针对知识召回优化会议中提到的
✂️切片/分块（Chunking） —— 知识库入库前必须做的步骤▶

一句话：把一篇长文档切成小段（如每段500字），方便向量化和检索。

为什么必须切：Embedding模型有字数限制（通常512或8192个token），太长的文档塞不进去。

会议中的关键讨论：

"切片实际上就是切片，就是你得把一个完整内容的段落切到一起，你不要把它切断了。就比方说法律第一条都没说完就切了，然后召回的知识是碎片的，或者有的时候就召不回来。"——这就是切片的核心难点：切的位置不对，检索质量就差。

常见切法：

  • 固定长度切：每500字切一段。简单粗暴，但可能把一句话切成两半
  • 按段落/章节切：以换行符或标题为界。更合理，但需要文档有结构
  • 智能切：用模型判断语义边界。最准确，但成本最高
🤖智能体（Agent） —— 不只会回答，还会行动▶

会议中："写智能体"、"开发一个智能体去过它处理完的文档"——就是做Agent。

🔧Dify —— 做智能体的低代码平台▶

一句话：一个开源的智能体开发平台，拖拽式配置，不需要写代码就能做AI应用。

会议中："在这个Dify的或者别的智能体开发框架上"——Dify就是你们要用的工作台。

Dify能做什么：

  • 配置知识库（上传文档→自动切片→向量化→入库）
  • 创建智能体（选模型+配提示词+挂知识库+加工具）
  • 设计工作流（拖拽节点串成流水线）
  • 发布应用（生成网页链接或API）

Dify分两个版本：

版本费用功能你们用哪个
开源版(Community)免费核心功能都有✅ 用这个
企业版(Enterprise)付费多一些管理功能不需要
💡 会议中提到的"百千万平台"就是在Dify开源版基础上二次开发的——加了一个应用市场，让大家可以共享和复用智能体。
🔄工作流（Workflow） vs Skill vs 超级智能体▶

这是会议中最核心的概念升级，三个层次：

第一代：工作流智能体

用户手动定义每一步做什么：第一步查法规、第二步比对、第三步生成报告。像流水线，固定路径。

优点：可控、快速、简单。缺点：只能做预设的事，遇到意外情况就卡住。

会议中："用户定义工作流的方式实现智能体"、"苏州公安做了600多个智能体"——都是工作流型。

第二代：Skill智能体

把AI的能力封装成一个个Skill（技能），智能体根据用户的请求自动选择调用哪个Skill。像工具箱，按需取用。

优点：灵活，可以应对不同场景。缺点：需要设计好Skill的接口。

会议中："真正执行的时候，其实还是写Skill，用Skill的方式去解决，这是目前比较好的落地形式"。

第三代：超级智能体（Super Agent）

智能体不仅能选工具，还能自动规划步骤、自我反思、多步推理。像一个能独立完成任务的实习生。

会议中："新一代的智能体是通过Skill的方式交互来形成的"、"现在其实大家就把它叫超级智能体"。

总结：工作流 = 固定流水线 | Skill = 按需调工具 | 超级智能体 = 自主规划+调工具
你们目前先用工作流把基础功能跑通，后续升级到Skill和超级智能体。
💬Cherry Studio —— 大模型的"遥控器"▶

一句话：一个本地安装的聊天软件，连接到你们内网的大模型，像用微信一样跟AI对话。

会议中："装一个Cherry Studio，它就是一个大模型的交互前端"、"把模型的token配置给到您，配到Cherry Studio里就可以"。

关键点：

  • 纯前端，不需要连互联网
  • 配置模型API地址和Token就能用
  • Windows版有安装包
  • ⚠️ 需要确认市局是否允许安装（白名单检查）
📝提示词（Prompt） —— 控制AI行为的"遥控器"▶

一句话：你告诉大模型"怎么回答问题"的指令文字。

会议中："你到时候一边用一边自己可以改提示词的编排"、"他检测的内容你来配"——提示词是你最核心的日常工作。

举个例子：

# 一个卷宗查错智能体的提示词示例
你是公安卷宗审核助手。请按照以下规则检查卷宗：
1. 检查所有文书日期是否逻辑一致（传唤时间≤拘留时间≤逮捕时间）
2. 检查嫌疑人姓名在各文书中是否一致
3. 检查强制措施期限是否超期
4. 检查处罚种类和幅度是否与违法事实匹配

输出格式：
- 问题描述
- 涉及文书名称
- 修改建议
- 依据的法律条文
📦Token —— AI世界的"计价单位"▶

一句话：大模型处理文字的最小单位，1个汉字≈1个token，英文单词≈0.75个token。

为什么重要：

  • 每次对话消耗token，token越多处理越慢、成本越高
  • 模型有上下文窗口限制（如128K token ≈ 10万字），超过就"记不住"
  • 卷宗很长（100页≈3万字），要注意是否超出模型窗口

💻 第四章：开发入门

从打开终端开始。你不需要会写代码，但必须能操作服务器、看懂命令、跟工程师对话。

🖥️本地电脑 vs 服务器 —— 先搞清楚"在哪台机器上"▶
本地电脑服务器
是什么你面前的笔记本/台式机机房里24小时开机的电脑
操作系统WindowsUbuntu Linux
用途日常办公、浏览网页运行AI系统、数据库
怎么用直接操作远程登录（SSH）

会议中的服务器：

  • 应用服务器：Ubuntu x86系统，跑Dify、知识库、数据库、文档存储
  • GPU服务器：带显卡的服务器，跑大模型推理（千问）
💡 为什么分两台？因为大模型推理需要GPU（贵），其他服务不需要GPU（便宜）。分开部署更灵活，GPU服务器以后要换也不影响应用服务器上的数据。
🔑SSH —— 远程进入服务器的"钥匙"▶

一句话：SSH是远程登录服务器的方式，让你在自己电脑上操作机房里的服务器。

怎么用（Windows电脑）：

# 方法1：Windows自带的终端（推荐）
按 Win+R → 输入 cmd → 回车
ssh 用户名@服务器IP地址
# 例如：
ssh admin@10.0.1.100
输入密码：******（密码不会显示，直接输完回车）
admin@app-server:~$  ← 出现这个就登录成功了！
# 方法2：用MobaXterm（图形化工具，更友好）
1. 下载安装 MobaXterm（免费版即可）
2. 点击 Session → SSH
3. 填入服务器IP、用户名
4. 点击连接，输入密码
⚠️ 你会需要的信息（提前问工程师要）：
• 服务器IP地址（如 10.0.1.100）
• 用户名（如 admin 或 root）
• 密码
• 端口号（默认22，如果有改的话）
🐧Linux 基础命令 —— 会这几条就够了▶

服务器用的是Ubuntu Linux，没有鼠标和窗口，全靠命令操作。以下是你必须会的命令：

# ===== 文件操作 =====
ls              # 列出当前目录的文件（像打开文件夹）
ls -la          # 列出所有文件（包括隐藏的），带详细信息

🐳Docker —— 应用打包的"集装箱"▶

一句话：把程序和它的运行环境打包成一个"集装箱"，搬到哪都能运行，不用操心环境配置。

为什么需要Docker：大模型系统涉及很多组件（模型服务、向量数据库、前端界面……），每个组件的环境要求不同。手动配置极其复杂，Docker一键搞定。

你需要知道的Docker命令：

docker ps                    # 查看正在运行的所有容器
docker ps -a                 # 查看所有容器（包括停止的）
docker logs 容器名            # 查看某个容器的日志（排错用）
docker restart 容器名         # 重启某个容器
docker stop 容器名            # 停止某个容器
docker start 容器名           # 启动某个容器
docker-compose up -d         # 启动整个系统（在docker-compose.yml所在目录）
docker-compose down          # 停止整个系统
💡 docker-compose是Docker的"指挥官"，一个配置文件定义所有容器怎么启动。工程师部署时会写好这个文件，你只需要知道up -d启动、down停止。
🌐端口和访问 —— 怎么在浏览器打开系统▶

一句话：服务器上的每个服务都有一个"门牌号"（端口号），浏览器通过 IP:端口 访问。

# 常见端口
http://10.0.1.100:80     # Dify前端界面
http://10.0.1.100:8000   # 大模型API服务
http://10.0.1.100:5432   # PostgreSQL数据库
http://10.0.1.100:19530  # Milvus向量数据库
⚠️ 防火墙必须开放对应端口，否则浏览器打不开。提前让信息中心开端口！
🔒信创 —— 公安IT环境的特殊约束▶

一句话：信创 = 信息技术应用创新，核心要求是用国产软硬件替代国外产品。

会议中："X86虚拟机"、"腾讯数据库"、"达梦数据库"——都涉及信创。

对你们的影响：

类别传统方案信创替代
操作系统Windows/Ubuntu麒麟/统信UOS
数据库MySQL/PostgreSQL达梦/TDSQL(腾讯)
CPUIntel/AMD (x86)鲲鹏/飞腾 (ARM)
💡 好消息：会议确认了你们的应用服务器是x86架构的Ubuntu，不是ARM信创环境，所以Docker镜像可以正常使用。数据库方面，如果可以用PostgreSQL就最简单。

3️⃣导入模型文件▶

模型文件通常几十GB（7B量化版约4-8GB，72B全量版约140GB），需要通过移动硬盘或内网传输导入。

⚠️ 会议提到："如果GPU服务器要换，文档要拷出来再同步就非常麻烦"——所以应用服务器硬盘要大，文档存应用服务器而不是GPU服务器。
